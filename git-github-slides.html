Why Version Control?

Frontend developers need to:

Track changes in HTML, CSS, JS files
Collaborate without overwriting code
Revert mistakes easily
Work on features safely

Git makes this possible!

What is Git?

A distributed version control system:

Tracks changes in your codebase
Fast, offline, and open-source
Used by 90%+ of developers

Created by Linus Torvalds (Linux kernel)

What is GitHub?

A platform for hosting Git repositories:

Cloud-based collaboration
Features: Pull requests, issues, Actions
Home to 100M+ repositories

Perfect for team projects at RAD5 Tech Hub!

Setting Up Git

Install Git: git-scm.com

Configure Git:

git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

Verify: git --version

Setting Up GitHub

Create an account at github.com

(Optional) Set up SSH:

ssh-keygen -t rsa -b 4096 -C "your.email@example.com"
cat ~/.ssh/id_rsa.pub

Add key to GitHub: Settings > SSH and GPG keys

Basic Git Workflow

Create a frontend project:

mkdir my-frontend-project
cd my-frontend-project
git init
touch index.html styles.css app.js
git add index.html
git commit -m "Add initial HTML structure"

Check progress: git status, git log

Pushing to GitHub

Create a repository on GitHub

Connect and push:

git remote add origin git@github.com:username/my-frontend-project.git
git branch -M main
git push -u origin main

Your code is now live on GitHub!

Collaborating with GitHub

Clone a repo:

git clone git@github.com:username/my-frontend-project.git

Add collaborators: Repository > Settings > Collaborators

Pull updates: git pull origin main

Branching

Create a branch for new features:

git checkout -b feature/add-button

Add changes, commit, and push:

git add .
git commit -m "Add button to homepage"
git push origin feature/add-button

Pull Requests

Propose and review changes:

Create a PR on GitHub for your branch
Team reviews and suggests changes
Merge into main branch

Update local repo: git checkout main; git pull origin main

Handling Conflicts

When two changes clash:

<<<<<<< HEAD
h1 { color: blue; }
=======
h1 { color: red; }
>>>>>>> feature/add-button

Resolve manually, then:

git add styles.css
git commit -m "Resolve merge conflict"
git push origin main

Best Practices for Frontend Devs

Commit often with clear messages
Use .gitignore for node_modules, .env
Create branches for features or fixes
Pull updates regularly to avoid conflicts
Use GitHub Actions for automation

Hands-On Project

Build a portfolio site with HTML, CSS, JS:

Create a GitHub repo
Add a collaborator
Create a branch for a feature (e.g., CSS animation)
Submit a PR and merge
Deploy with GitHub Pages
